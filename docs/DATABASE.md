# Database Schema Documentation

## Overview

BeatForge uses PostgreSQL for persistent data storage. The schema is designed to support:
- Song upload tracking
- Job/generation history
- Beatmap metadata
- Detailed logging for debugging

## Schema Diagram

```
Songs (1) ──── (many) Jobs
   │                  │
   │                  └──── (1) Beatmaps
   │
   └──── (many) JobLogs
```

## Tables

### Songs Table

Stores information about uploaded MP3 files.

```sql
CREATE TABLE songs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  filename VARCHAR(255) NOT NULL,
  original_filename VARCHAR(255),
  duration FLOAT NOT NULL,
  file_size BIGINT NOT NULL,
  file_path VARCHAR(512),
  format VARCHAR(20),  -- 'mp3', 'wav', 'ogg'
  sample_rate INTEGER,
  channels INTEGER,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP,
  
  CONSTRAINT file_size_check CHECK (file_size > 0),
  CONSTRAINT duration_check CHECK (duration > 0)
);

CREATE INDEX idx_songs_created_at ON songs(created_at);
CREATE INDEX idx_songs_deleted_at ON songs(deleted_at) WHERE deleted_at IS NULL;
```

**Columns:**
- `id`: Unique identifier (UUID v4)
- `filename`: Stored filename (hash-based for security)
- `original_filename`: User-provided filename
- `duration`: Song length in seconds
- `file_size`: File size in bytes
- `file_path`: Path to file in storage
- `format`: Audio format
- `sample_rate`: Audio sample rate (Hz)
- `channels`: Mono (1) or stereo (2)
- `created_at`: Upload timestamp
- `updated_at`: Last modification
- `deleted_at`: Soft delete timestamp

### Jobs Table

Tracks generation jobs and their status.

```sql
CREATE TABLE jobs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  song_id UUID NOT NULL REFERENCES songs(id) ON DELETE CASCADE,
  status VARCHAR(50) NOT NULL DEFAULT 'QUEUED',
  difficulty VARCHAR(50) NOT NULL,
  mapping_style VARCHAR(50) NOT NULL,
  target_star_rating FLOAT,
  progress INTEGER DEFAULT 0,
  current_stage VARCHAR(255),
  result JSONB,
  error_message TEXT,
  error_traceback TEXT,
  celery_task_id VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  started_at TIMESTAMP,
  completed_at TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT progress_check CHECK (progress >= 0 AND progress <= 100),
  CONSTRAINT star_rating_check CHECK (target_star_rating IS NULL OR (target_star_rating >= 2 AND target_star_rating <= 9))
);

CREATE INDEX idx_jobs_song_id ON jobs(song_id);
CREATE INDEX idx_jobs_status ON jobs(status);
CREATE INDEX idx_jobs_created_at ON jobs(created_at DESC);
CREATE INDEX idx_jobs_celery_task_id ON jobs(celery_task_id);
```

**Status Values:**
- `QUEUED`: Waiting in job queue
- `VALIDATING`: Validating input file
- `ANALYZING`: Analyzing audio features
- `GENERATING`: Generating beatmap
- `PACKAGING`: Creating .osz file
- `COMPLETED`: Successfully generated
- `FAILED`: Generation failed
- `CANCELLED`: User cancelled job

**Columns:**
- `id`: Job identifier
- `song_id`: Reference to uploaded song
- `status`: Current job status
- `difficulty`: Requested difficulty (EASY, NORMAL, HARD, INSANE, EXPERT_PLUS)
- `mapping_style`: Requested style (TECHNICAL_STREAM, JUMP, HYBRID, AIM, STREAM_PRACTICE)
- `target_star_rating`: Requested difficulty in stars (2-9)
- `progress`: 0-100 completion percentage
- `current_stage`: User-friendly stage description
- `result`: JSON with beatmap metadata (see below)
- `error_message`: User-friendly error message
- `error_traceback`: Full Python traceback for debugging
- `celery_task_id`: Celery task identifier for monitoring
- `created_at`: Job creation time
- `started_at`: When processing started
- `completed_at`: When processing finished
- `updated_at`: Last status update

**Result JSON Structure:**
```json
{
  "beatmap_id": "uuid",
  "file_path": "/storage/beatmaps/uuid.osz",
  "bpm": 170,
  "estimated_star_rating": 5.2,
  "object_count": 523,
  "drain_time": 180,
  "total_time": 185,
  "circle_count": 245,
  "slider_count": 200,
  "spinner_count": 3,
  "combo_colors": [[255, 0, 0], [0, 255, 0], [0, 0, 255]],
  "creator": "BeatForge AI",
  "version": "Generated by BeatForge v1.0"
}
```

### Beatmaps Table

Stores generated beatmap metadata.

```sql
CREATE TABLE beatmaps (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id UUID NOT NULL REFERENCES jobs(id) ON DELETE CASCADE,
  song_id UUID NOT NULL REFERENCES songs(id) ON DELETE CASCADE,
  file_path VARCHAR(512) NOT NULL,
  osz_file_path VARCHAR(512),
  osu_file_path VARCHAR(512),
  
  metadata JSONB,
  
  difficulty_rating FLOAT,
  object_count INTEGER,
  circle_count INTEGER,
  slider_count INTEGER,
  spinner_count INTEGER,
  
  bpm FLOAT,
  drain_time FLOAT,
  total_time FLOAT,
  
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP,
  
  CONSTRAINT object_count_check CHECK (object_count > 0)
);

CREATE INDEX idx_beatmaps_job_id ON beatmaps(job_id);
CREATE INDEX idx_beatmaps_song_id ON beatmaps(song_id);
CREATE INDEX idx_beatmaps_created_at ON beatmaps(created_at);
```

**Metadata JSON Structure:**
```json
{
  "title": "Song Title",
  "artist": "Artist Name",
  "creator": "BeatForge AI",
  "version": "[BeatForge] Normal",
  "source": "osu!",
  "tags": ["generated", "ai", "beatforge"],
  "combo_colors": [[255, 0, 0], [0, 255, 0]],
  "audio_file": "audio.mp3",
  "background_file": null
}
```

### JobLogs Table

Detailed logs for each job for debugging and monitoring.

```sql
CREATE TABLE job_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id UUID NOT NULL REFERENCES jobs(id) ON DELETE CASCADE,
  timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  level VARCHAR(20),  -- DEBUG, INFO, WARNING, ERROR, CRITICAL
  message TEXT NOT NULL,
  context JSONB,
  
  CONSTRAINT level_check CHECK (level IN ('DEBUG', 'INFO', 'WARNING', 'ERROR', 'CRITICAL'))
);

CREATE INDEX idx_job_logs_job_id ON job_logs(job_id);
CREATE INDEX idx_job_logs_timestamp ON job_logs(timestamp DESC);
CREATE INDEX idx_job_logs_level ON job_logs(level);
```

**Context JSON Example:**
```json
{
  "stage": "bpm_detection",
  "duration": 0.234,
  "result": {
    "bpm": 170.5,
    "confidence": 0.95
  }
}
```

## Data Lifecycle

### Song Upload

1. File validated (size, format, duration)
2. File hashed and stored
3. Audio metadata extracted
4. Song record created
5. User receives `song_id`

### Generation Job

1. Job created in QUEUED status
2. Celery task enqueued
3. Worker picks up task
4. Status updated to ANALYZING
5. Audio features computed and cached
6. Status updated to GENERATING
7. Beatmap generated
8. Status updated to PACKAGING
9. .osz file created
10. Beatmap record created
11. Status updated to COMPLETED
12. Result JSON populated
13. Temporary files cleaned up

### Cleanup

After 30 days, completed jobs can be archived:

```sql
ARCHIVE jobs WHERE completed_at < NOW() - INTERVAL '30 days' AND status = 'COMPLETED';
```

After 7 days, failed jobs are deleted:

```sql
DELETE FROM jobs WHERE completed_at < NOW() - INTERVAL '7 days' AND status = 'FAILED';
```

## Queries

### Get recent jobs

```sql
SELECT 
  j.id,
  j.status,
  j.difficulty,
  j.progress,
  s.original_filename,
  j.created_at
FROM jobs j
JOIN songs s ON j.song_id = s.id
WHERE j.created_at > NOW() - INTERVAL '24 hours'
ORDER BY j.created_at DESC;
```

### Get job completion rate

```sql
SELECT 
  status,
  COUNT(*) as count,
  ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER (), 2) as percentage
FROM jobs
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY status;
```

### Get average generation time

```sql
SELECT 
  difficulty,
  COUNT(*) as count,
  ROUND(AVG(EXTRACT(EPOCH FROM (completed_at - started_at)))::numeric, 2) as avg_seconds
FROM jobs
WHERE status = 'COMPLETED'
  AND completed_at > NOW() - INTERVAL '7 days'
GROUP BY difficulty;
```
